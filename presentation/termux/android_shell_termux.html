Installation

From Termux FAQ: Termux is a terminal emulator for Android. The key difference from other terminal emulators is its own package repository with huge set of various utilities.

Can be downloaded from F-Droid or Google Play.

What can it do?

  • bash/zsh
  • coreutils (ls, tar, find…)
  • curses & colors ("graphics")

bash (busybox)

Zsh available, if that's your thing.

coreutils

Full GNU tools available. dd, and all!

UTF-8 support! Type weird letters, make dialog-style boxes! Math!

Colors support!

256 colors for sure via xterm-256color.

Termux claims 24 bit is supported too, but I cannot prove that: it's surprisingly hard to get an application to use it, and 256 colors was already good enough for me

curses

Curses is a library that enables taking over the terminal to make a full canvas (as opposed to purely request and response).

Mouse support! htop is famous for the support of mouse.

SSH

server & client

  • Wifi
  • USB
  • Bluetooth

Connect to your desktop/server from phone, or get them to connect to you!

Works (tested) over wifi, bluetooth, USB…

Dotfiles sync

Bluetooth file exchange to sync git repos workâ†”phoneâ†”home

  • Git bundle to file
  • Bluetooth send
  • unpack as git remote

I wanted to try syncing my config files without a cloud like Github.

My phone is technically doing the a shuttle between work and home every day. I could use it as a USB drive to move git commits across and that would be it…

But I kind of want these same config files on my phone, since all my dev env is there too!

So instead, I use bluetooth file-sending (OBEX) to exchange git commits (git bundles), and unpack them on the other side as a git remote I can pull from (read-only).

Overkill, but really fun, and I learned something.

Limitations

Termux is not Linux:

  • Not FHS (no /usr/)
  • Root FS is normal data
  • single user (Android app limitation)

Stolen shamelessly from Termux wiki. Look into termux-chroot, which might work for some apps.
